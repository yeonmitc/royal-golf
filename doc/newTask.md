# 📅 직원 교대근무 스케줄러 UI/UX 및 DB 개발 사양서 (v2)

본 사양서는 기존 Production DB 스키마(`employees`, `attendance_logs`)를 기반으로, 웹서비스 내에서 작동할 파트타임 직원(3명 중 일일 1~2명) 달력 스케줄러의 UI/UX 레이아웃과 데이터 모델링 연계 방안을 정의합니다.

---

## 1. 💾 데이터베이스(DB) 구조 및 연계 가이드

기존 `attendance_logs`는 출근 도장을 찍을 때 생성되는 **실제 근태 기록**이므로, 달력에 미리 일정을 배정하고 조회하기 위한 **'스케줄 예약 전용 테이블'**을 아래와 같이 추가하여 연계합니다.

### 🆕 추가 테이블: `employee_schedules` (직원 스케줄 배정)

관리자가 달력에 직원을 드래그 앤 드롭할 때 데이터가 쌓이는 테이블입니다.

| Column Name   | Type          | Constraints                  | Description                      |
| :------------ | :------------ | :--------------------------- | :------------------------------- |
| `id`          | `int8`        | Primary Identity             | 스케줄 고유 고유 ID              |
| `employee_id` | `uuid`        | Foreign Key (`employees.id`) | 배정된 직원의 UUID               |
| `shift_type`  | `text`        | Not Null                     | `6AM` 또는 `9AM` (선택된 시프트) |
| `work_date`   | `date`        | Not Null                     | 근무 배정 날짜 (YYYY-MM-DD)      |
| `created_at`  | `timestamptz` | Default `now()`              | 스케줄 생성 일시                 |

> 🔒 **데이터 무결성 제약 조건 (Unique Constraint)**
>
> - `UNIQUE (employee_id, work_date)`: 한 직원이 같은 날짜에 `6AM`과 `9AM`에 중복 배정되는 것을 데이터베이스 레벨에서 차단합니다.

### 🔄 기존 테이블과의 데이터 흐름 (Data Flow)

1. **스케줄링 시:** `employees` 테이블의 `id`와 `english_name`을 읽어와 화면에 카드로 노출 ➡️ 드래그 시 `employee_schedules`에 저장.
2. **출근 체크 시:** 직원이 모바일로 출근 버튼을 누르면 `attendance_logs` 테이블에 로그가 생성되며, `shift_type`과 비교하여 `is_tardy` (지각 여부)를 판별하는 로직으로 확장 가능.

## 2. 📱 모바일 직원 모바일 뷰 (조회 전용 UI)

직원들이 본인의 스마트폰(모바일 웹 브라우저) 환경에서 본인의 스케줄을 한눈에 확인할 수 있는 화면입니다.

### 💡 주요 UI/UX 요구사항

- **반응형 캘린더 (Mobile-First):** 화면 해상도에 맞춰 캘린더 셀 크기가 유연하게 조절되는 뷰.
- **직관적인 시프트 태그 (Shift Badge):** \* 달력 내부 날짜 칸에 배정된 직원의 `english_name`과 `shift_type`이 컬러 배지로 표시됩니다.
  - **[6AM] Min-su** (예: 파란색) / **[9AM] Ji-won** (예: 주황색)
- **내 일정 필터링:** 상단의 `[내 일정만 보기]` 토글 버튼을 켜면, 로그인한 직원의 일정만 하이라이트되고 타 직원의 일정은 반투명하게 처리됩니다.
- **하단 상세 팝업 (Bottom Sheet):** 모바일 특성상 캘린더 안의 좁은 칸을 터치하면 화면 하단에서 슬라이드가 올라오며 당일 근무 정보 및 동료 파트타이머를 시각적으로 보여줍니다.

## 3. 🛡️ 관리자 PC 웹 뷰 (드래그 앤 드롭 배정 UI)

데스크톱 브라우저 환경에서 빠르고 직관적으로 교대근무를 설정할 수 있는 화면입니다.

### 💡 컴포넌트 레이아웃 (2-Column)

- **좌측 컨트롤 패널 (Admin Panel):**
  1. **시프트 토글 (Radio Button):** `[6AM Shift]` 또는 `[9AM Shift]` 중 현재 배정할 타임라인을 먼저 라디오 버튼으로 선택합니다.
  2. **직원 카드 목록 (Draggable List):** DB의 `employees` 테이블에서 가져온 리스트를 카드 형태로 나열합니다. 카드 컴포넌트 내부에는 `employees.id` 데이터가 포함되어 있습니다.
- **우측 대형 달력 (Grid Calendar):** 한 달 단위의 메인 스케줄러 그리드 인터페이스입니다.

### 🔄 드래그 앤 드롭 동작 로직 (D&D Workflow)

1. **[시프트 선택]** 관리자가 `6AM Shift` 라디오 버튼을 클릭합니다.
2. **[직원 드래그]** `employees` 리스트에서 특정 직원 카드를 마우스로 클릭 및 홀드하여 달력 영역으로 드래그합니다.
3. **[날짜 드롭]** 원하는 날짜 그리드(Drop Zone) 위에서 마우스를 놓습니다.
4. **[API 연동]** 프론트엔드에서 드롭된 `날짜(work_date)`, 선택되어 있던 시프트 `6AM(shift_type)`, 카드의 `직원 ID(employee_id)`를 엮어 `employee_schedules` 테이블에 즉시 `INSERT` 요청을 보냅니다.
5. **[실시간 반영]** 성공 시 달력 셀 내부에 `[6AM] english_name` 배지가 렌더링됩니다.

### ⚠️ 비즈니스 밸리데이션 (Validation Rules)

- **일일 정원 제한:** 하루에 근무 가능한 파트타임 인원은 **최대 2명**입니다. 날짜 셀에 이미 2개의 스케줄이 존재할 경우 드롭 동작을 무효화하고 경고 알림(Toast)을 노출합니다.
- **삭제 및 수정:** 달력에 등록된 직원 배지 우측의 `X` 버튼을 누르거나 마우스 우클릭 시, 해당 스케줄 `id`를 기준으로 테이블에서 `DELETE` 쿼리가 실행되어 일정이 취소됩니다.
